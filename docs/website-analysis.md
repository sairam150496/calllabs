## Design Specifications

### Typography
- **Main Heading**: Large, bold, green
- **Subheadings**: Medium weight, black/dark gray
- **Body Text**: Regular weight, gray
- **Price Text**: Large, bold, green/purple/red depending on package
- **Buttons**: White text on colored background

### Spacing & Layout
- **Container Width**: ~1200px max-width
- **Grid System**:
  - Services: 4 columns (desktop)
  - Packages: 3 columns with featured card
  - How It Works: 4 columns
  - Footer: 4-5 columns

### Button Styles
- **Primary (Green)**: "Book Test", "Book Now"
- **Secondary (Red)**: "Call Ambulance", "Call Now", "Emergency"
- **Tertiary (Purple)**: Advanced services
- **Outline**: "View Packages", "Consult Now", etc.
- **Shape**: Rounded corners (~4-8px border-radius)

### Icons
- Line icons with circular backgrounds
- Service icons use brand colors
- Trust badges use simple iconography
- All icons appear to be custom or from a cohesive icon set

### Images
- Hero image: Professional photography with healthcare theme
- Ambulance: Product/service photography
- Background: Subtle city skyline
- All images have clean, professional aesthetic

### Responsive Breakpoints
- **Desktop**: 1200px+
- **Tablet**: 768px - 1199px
- **Mobile**: < 768px

### Responsive Behavior
- Mobile navigation (hamburger menu)
- Service grid: 4 → 2 → 1 column
- Packages: 3 → 2 → 1 column
- Hero: Stack text and image vertically on mobile
- Footer: 4-5 columns → 2 columns → 1 column
- Buttons: Full width on mobile
- Font sizes: Scale down appropriately

## Mobile-Specific Features

### Sticky Elements
- **Floating Emergency Button**: Bottom right, red, circular with ambulance icon
- **Floating WhatsApp Button**: Bottom left, green, circular with WhatsApp icon
- **Header**: Sticky on scroll (optional)

### Touch Interactions
- **Tap Targets**: Minimum 44x44px for all buttons
- **Swipe Gestures**: For service carousel and packages
- **Pull to Refresh**: Homepage (optional)
- **Click to Call**: Phone numbers
- **Click to WhatsApp**: WhatsApp button opens chat

### Performance Optimizations
- **Lazy Loading**: Images load as user scrolls
- **Compressed Images**: WebP format, optimized sizes
- **Reduced Animations**: Minimal animations for performance
- **Fast Loading**: Critical CSS inline, defer non-critical

## Mobile Typography Scale

### Font Sizes
- **H1 (Main Heading)**: 32px - 36px (down from 48px+)
- **H2 (Section Titles)**: 24px - 28px
- **H3 (Card Titles)**: 18px - 20px
- **Body Text**: 14px - 16px
- **Button Text**: 16px (readable, tappable)
- **Small Text**: 12px - 14px

### Line Heights
- Headings: 1.2 - 1.3
- Body: 1.5 - 1.6
- Buttons: 1.2

## Mobile Spacing System

### Padding
- **Container**: 16px horizontal
- **Cards**: 16px all sides
- **Sections**: 40px - 60px vertical
- **Elements**: 8px - 16px gaps

### Margins
- **Between Sections**: 40px
- **Between Cards**: 12px
- **Between Buttons**: 12px

### Requirements
- Touch targets ≥ 44x44px

## Mobile Loading States

### Skeleton Screens
- Show content placeholders while loading
- Maintain layout structure
- Smooth transition to actual content

### Progressive Enhancement
- Core content loads first
- Enhanced features load after
- Graceful degradation for older devices

## Mobile Performance Targets

### Metrics
- **First Contentful Paint**: < 1.5s
- **Largest Contentful Paint**: < 2.5s
- **Time to Interactive**: < 3.5s
- **Cumulative Layout Shift**: < 0.1
- **Page Size**: < 2MB total

## Mobile Image Optimization

### Strategies
- Responsive images with srcset
- WebP with JPEG fallback
- Lazy loading for below-fold images
- Proper sizing (no oversized images)
- CDN delivery
